nodeJs笔记

node.js是什么

node.js是一个js运行时环境+大量的新API
node.js = 谷歌V8引擎 + 大量新API
使用nodejs来编写服务端代码

node.js搭建服务器的好处
前端人员熟悉JS语法
效率非常高，是普通服务器的10倍
敏捷开发

node的安装

http://nodejs.cn
下载->windows 64位.msi
打开控制台:windows键+R->cmd
node -v 目的是为了验证node有没有安装,并不是为了关注版本号

第一个nodejs程序

node 文件名
注意:请将文件的目录切换至操作目录

node搭建服务器

//创建可一个http对象
const http = require('http');
//创建一个服务器对象
http.createServer(function(req, res) {
//设置响应头,支持中文,将响应的字符串以HTML的形式解析
res.writeHead(200, { "Content-type": "text/html;charset=utf-8" });
res.write("hahaha");
res.write("xixixi");
//end([响应给用户的内容]);
res.end("<strong>王力宏</strong>");
//设置端口号 0~65535   8500以上

}).listen(8888);

nodejs中的模块化

以前js文件相互交互是通过HTML文件实现的

nodeJS文件导入和导出的规则

模块:在node中,所有的js文件都被称为模块（本质和类是差不多的）
模块的分类
1.系统模块 http,fs
2.自定义模块
3.第三方模块 (通过软件商城下载的模块)

CommonJS:node实现js文件引入js文件的规则
导入:
    const myModule = require(模块名称);
导出:
    可出现多次
    exports.属性名 = 属性

    只能出现一次
    module.exports = {
        属性1:值1,
        属性2:值2...
    }

//模块的组合
module.exports = {
    key1: myModule,
    fun3() {
        console.log("fun3");
    }
}

node_modules：
存放在该文件夹的模块,在导入的时候不需要添加路径

ES6的模块化:前端js文件引js文件的语法规则
导出:
    方法1
    export let a = 666;
    export function fun(a, b) {
        return a + b;
    }
    方法2
    let a = 888;
    let b = 999;

    function fun() {
        console.log("fun");
    }

    export default {
        a,
        fun
    }

导入:
    方法1
    import { a, fun } from "./3.a.js";

    方法2
    import myModule from "./3.a.js";
    console.log(myModule.a);
    myModule.fun();

最后需要在html文件中,添加type="module"属性

npm

包管理工具:用来下载第三方模块

目标:会使用npm下载第三方模块

1.下载模块
npm install 模块名称
npm i 模块名称

2.卸载模块
npm uninstall 模块名称

3.查看node_modules安装的包
npm list

4.查看安装的某个包的详细信息
npm info 模块名称

npm install -g  nodemon 自动重启工具
