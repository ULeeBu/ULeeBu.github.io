hexo博客搭建梳理

起因：

-之前装过几次博客，最后都因为没东西写而荒废，即便有东西写也都在csdn上写着，
近来由于习得git和nodeJs，故想起这个博客都有用到这两种工具，便又重新拾起，
这篇文章简单梳理下hexo的安装方法，以便自己后续查阅

搭建环境

-安装nodeJs(用来生成hexo框架)
-git(用来管理博客)
-github或者gitee账号(各有优缺点，gitee速度快，但是需要开启git page, 开启的时候要提交身份证；github访问速度慢点)

步骤

首先使用nodeJs安装hexo框架
`npm install hexo-cli g`
安装好之后再创建博客文件夹，进行博客初始化，生成本地博客
`hexo init`
生成之后我们就可以使用下面的命令先在本地启动查看样式,后面进行博客更新都可以先在本地启动看看
`hexo s`
上面命令执行之后就可以在浏览器打开localhost:4000查看，启动失败大概率是端口被占用
可以使用以下来进行更改端口号启动
`hexo s -p 端口号`
启动成功之后接下来我们就可以把博客部署到远端了
部署到远端我们需要安装一个deployer的插件
`npm install hexo-deployer-git --save`
安装好之后接下来我们去GitHub或者gitee上操作
首先创建一个仓库，仓库名一定要与你的账号名称一致，按照这样的格式填写xxxx.github.io **切记！！**
创建好之后复制仓库链接，打开博客目录下的_config.yml文件拉到最底下
如下填写，repo后面改为博客的仓库地址
`deploy:
type: 'git'
repo: git@github.com:xxxx/xxxx.github.io.git
branch: master`
修改完毕之后回到git
使用`hexo g -d`将本地页面部署到远端
地址栏输入 xxxx.github.io 打开则部署成功
由于hexo博客没有后台管理，写的博客在source/_posts目录下
每次写完再部署下即可更新，主题安装可以在github上搜索，一般都会有安装步骤
